## Tips for Information Architecture

1. **Site Map**
   - Keep it under 7 main categories (Miller's Law)
   - Use card sorting with real users to validate
   - Mark P0/P1/P2 priorities for phased rollout
   - Update as features evolve

2. **Navigation**
   - Limit primary navigation to 5-7 items
   - Mobile: Bottom tab bar for 4-5 core functions
   - Provide search for deep content
   - Show current location (breadcrumbs, active states)

3. **Content Hierarchy**
   - Follow F-pattern/Z-pattern reading behaviors
   - Place CTA above the fold
   - Use inverted pyramid (important first)
   - Test with 5-second test (what do users remember?)

4. **Taxonomy**
   - Design for growth (add new categories easily)
   - Allow multi-dimensional classification
   - Use user language, not internal jargon
   - Test search queries to validate tag relevance
